pyFormex is a python implementation of Formex algebra.
pyFormex lets you generate large geometrical models of 3D structures by a sequence of mathematical transformations.
It is especially suited for the automated design of spatial frame structures. But as the examples show,
it can also be used for other tasks, like finite element preprocessing, or just for creating some nice pictures.

Prerequisites

For running pyFormex, you need to have the following packages installed (and working):

  • Python (version 2.2 or higher) is used as the basic development language and the scripting language in pyFormex.
  • NumPy (version 0.9.4 or higher) is the underlying numerical array package.
    (Versions of pyFormex before 0.3 use an older array package, numarray, instead of numpy!)
  • PyOpenGL (version 2.0 or higher) is needed for the 3D rendering of the generated structures.
  • PyQt (version 0.9 or higher) was choosen for building the Graphical User Interface.
    (Obviously, this means that you also need a working Qt3)

If all you want to do is use the pyFormex mathematical transformations, the first two packages should be sufficient.
If you want to use the graphical interface and create images, you will also need the other two.

Platform

While pyFormex is developed on Linux platforms, it should be possible to get it running on other platforms too, provided you have installed the above packages.

Optional packages

  • If you want to save the images in .PS, .EPS or .PDF format (as shown in the WireStent example), you need the gl2ps OpenGL to PostScript printing library.
  • If you want to perform the structural analysis of the truss in the X_truss_analysis example, you need the calpy analysis package version 0.3.2 or higher. Install by preference from the source; compiling should work without problems if you have a full NumPy installation.

Documentation

Meanwhile, you can learn a lot by studying the examples and the comments in the source code.
